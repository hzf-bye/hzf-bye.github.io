mybatis基础知识

1.mybatis是什么

  1. mybatis是一个优秀的基于java的持久层框架，它内部封装了jdbc，使开发者只需要关注sql语句本身，而不需要花费精力去处理加载驱动、创建连接、创建statement等繁杂的过程。
  2. mabatis让程序将主要精力放在sql上，通过mybatis提供的映射方式，自动灵活生成（半自动化大部分需要程序员编写sql）满足需要sql语句。
  3. mybatis可以将preparedStatement中的参数自动进行输出映射，将查询结果集灵活映射成java对象（输出映射）。
  4. mybatis通过xml或注解的方式将要执行的各种statement配置起来，并通过java对象和statement中sql的动态参数进行映射生成最终执行的sql语句，最后由mybatis框架执行sql并将结果映射为java对象并返回。

2.mybatis框架

  1. SqlMapConfig.xml(是mybatis全局配置文件 名称不固定的)配置了运行环境，数据源，事务等mybatis运行环境

配置了映射文件（配置sql语句）

mapper.xml（配置文件）

2.SqlSessionFactory（会话工厂）

作用：创建SqlSession

3.SqlSession(会话) 面向用户的接口

作用：操作数据库（发出sql 增删改查）

4.Excutor(执行器)是一个接口（基本执行器，缓存执行器）

作用：SqlSession内部通过执行器操作数据库

5.mappedStatement (底层封装对象)

作用：对数据库存储封装，包括sql语句，数据参数，输出结果类型。

自增主键的返回

mysql自增主键，执行insert提交之前自动生成一个自增主键

通过mysql函数获取到刚刚插入记录的自增主键：LAST_INSERT_ID()

insert之后调用此函数

代码：

<!-- 
     parameterType:指定输入参数类型数pojo(包括用户信息)
     #{}中指定pojo的属性名，接受到pojo对象的属性值，mybatis通过OGNL获取对象的属性值。
 -->
<insert id="insertUser" parameterType="com.hzf.mybatis.po.User">

    <!-- 
    将插入数据的主键返回至user对象中
    SELECT LAST_INSERT_ID() 得到刚insert进去的主键值，只适用于主键自增时
    keyProperty：将查询到的主键值记录到parameterType指定的对象的对应属性中
    order：SELECT LAST_INSERT_ID()的执行顺序，相对于insert语句来说它的执行顺序
     -->
    <selectKey keyProperty="id" order="AFTER" resultType="java.lang.Integer">
        SELECT LAST_INSERT_ID()
    </selectKey>
    insert into user(username,birthday,sex,address) value(#{username},#{birthday},#{sex},#{address})
</insert>

9.总结

  1. selectOne表示查询一条记录进行映射，如果使用selectOne可以实现，使用selectList也可以实现（list中只有一条记录）
  2. selectList表示查询一个列表（多条记录）进行映射，如果使用selectList不能使用selectOne。

#{}与${}

#{}表示一个站位符，#{}接受输入参数，类型可以死简单类型，pojo，hashmap

如果接受简单类型，#{}中可以写成value或者其他名称

#{}接受pojo对象，通过OGNL读取对象中的属性值，通过属性.属性.属性…的方式获取对象属性值。

${}表示一个拼接符，将接受到的参数内容，不加任何修饰的拼接到sql中，

\${}会引起sql注入，所以不建议使用

${}接受输入参数，类型可以死简单类型，pojo，hashmap

如果接受简单类型，${}中只能写成value

${}接受pojo对象，通过OGNL读取对象中的属性值，通过属性.属性.属性…的方式获取对象属性值。
